nginx神奇的斜线

来自：https://blog.csdn.net/scdxmoe/article/details/52512143

来自：https://www.cnblogs.com/007sx/p/7151720.html

同样的坑：http://ju.outofmemory.cn/entry/374096

访问的URL：http://192.168.1.4/proxy/test.html

第一种：

location  /proxy/ {
          proxy_pass http://127.0.0.1:81/;
}

被代理到：http://127.0.0.1:81/test.html

第二种（相对于第一种，后面少了一个/）

location  /proxy/ {
          proxy_pass http://127.0.0.1:81;
}

会被代理到http://127.0.0.1:81/proxy/test.html 这个url

第三种

location  /proxy/ {
          proxy_pass http://127.0.0.1:81/ftlynx/;
}

会被代理到http://127.0.0.1:81/ftlynx/test.html 这个url。

第四种（(相对于第三种，最后少一个 / )）

location  /proxy/ {
          proxy_pass http://127.0.0.1:81/ftlynx;
}

会被代理到http://127.0.0.1:81/ftlynxtest.html 这个url
